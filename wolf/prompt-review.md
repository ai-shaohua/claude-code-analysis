# AI 狼人杀 Prompt 点评与优化建议

> 针对 `sample-context.md`(12人局-狼王守卫,4号猎人首轮发言)的分析。
> 前提约束:无工具调用、一次性全量上下文推理、不引入复杂信念系统与大量硬编码。

## 总体判断

这套 prompt 的**信息是够的,但组织方式在跟模型的推理能力抢资源**。很多"效果不理想"其实不是模型不行,而是 prompt 让模型把算力花在了解析散文、消化重复内容、以及在互斥指令间摇摆上,而不是推理本身。

估计当前"效果不理想"里,一多半是 prompt 可解的(结构、矛盾、记牌方式),剩下的才是模型对抗性推理的真实上限。先把可解的清掉,再判断模型够不够。

---

## 一、最影响"读懂"的结构问题(高性价比,先改)

### 1. 强调符号通货膨胀 —— 最大的问题
`⚠️最高优先级`、`【】`、`**加粗**`、`必须/禁止/严格`、`超过200字视为失败`…… 几乎每段都在喊。

Claude Code 官方 system prompt 里 `IMPORTANT` 用得极克制,正因为**当所有东西都是最高优先级时,模型就无法排序**,会平摊注意力,反而抓不住真正的铁律。

- **改法**:全篇只保留 2–3 处真正的硬约束(身份声称一致性、输出格式),其余降级为普通陈述。

### 2. 同一规则重复多遍
- 警长竞选结果:第 20–30 行、第 77–80 行各写一次,正文还引用。
- "单边"定义:第 58、66、68 行反复定义。
- 警长权力/审查:第 34、53–55 行分散。

重复不会强化,只会拉长上下文、增加"这两处是不是一回事"的判断负担。**每个事实/规则只说一次,放在唯一位置。**

### 3. 分节标题层级混乱
`## 当前局势` 下紧跟 `## 游戏历史事件`(实为父子却同级);`## 角色性格`/`## 思维方式`/`## 好人阵营行动准则` 夹在游戏状态中间。模型要靠语义猜哪些是"规则"、哪些是"本局数据"。应用固定三大类,顺序永远一致(见第四节)。

---

## 二、最影响"推理质量"的问题(核心)

### 1. 把"结论/策略先验"当成"推理指令",压制了推理
第 49 行这类**双向都成立**的表述:
> 大票型未必正确,更可能狼队冲票;小票型未必带团队,可能为干净好人票。

告诉模型"A 可能、B 也可能"等于没给信息,却占注意力,还诱导"两边都提一句"的空话。

- **改法**:从"规则"改成**开放式提问**,逼模型 grounding。例如:
  > 3 号得 7 票当选,这个票型是好人自然归票,还是狼队冲票做警?请结合投票人身份线索判断。

  **推理质量来自"被要求回答具体问题",不是来自"被告知答案"。**

### 2. 硬规则与"独立判断"直接打架
- 第 68 行:单边神职出现时 **"必须站边……不能质疑……不能投票"**
- 第 50 行:**"永不空口说 X 是好人/狼人""敢于承认逻辑盲区"**
- 第 53 行:警长 **"不一定是好人……须独立判断"**

一边"必须无条件站边",一边"必须独立判断、绝不盲信",模型在两者间摇摆,发言常自相矛盾。

- **改法**:想清楚单边神职保护是**铁律**还是**倾向**,二选一。建议改成倾向性表述(提高攻击/出局门槛),而非"必须/不能"。

### 3. persona 太重,且与推理逻辑缠在一起
第 44–46 行性格("冷静如冰""无奈轻叹""拆解至溃")是**说话风格**,却被写进推理链路(紧接第 49–50 行"思维方式")。模型分不清"该怎么想"还是"该怎么演",容易为演性格牺牲判断。

- **改法**:**把"怎么推理"和"怎么说话"物理分开**。推理框架对所有角色通用、干净;persona 只在"发言风格"节生效,只管语气/措辞。

### 4. 用散文记牌,模型得先"读懂局面"才能推理 —— 最有效的单一杠杆
现在历史是半结构化散文(第 22–30 行)。模型推理前要先在脑内重建"谁跳什么、谁验谁、谁投谁",耗算力且易错。你已有完整引擎数据,直接喂结构化摘要:

```
## 身份声称(claims)
7号→预言家: 查杀3号 | 警徽流8,11
8号→预言家: 金水10 | 警徽流12,4
3号→预言家: 金水12 | 警徽流4,9  (当选警长)
→ 三预对跳,真预言家仅1位

## 投票矩阵(警长竞选)
3号 ← 1,2,4,6,9,10,12 (7票)
(其余无票)

## 存活/死亡
全12人存活;第1夜平安夜
```

这不是复杂信念系统,只是把已有数据换个排版。**模型读表比读散文准得多。**

## 三、建议的重组骨架(顺序固定:最稳定 → 最易变)

```
[A. 游戏规则与配置]      ← 整局不变,可跨局复用
   - 局配置、胜负条件、机制(同守同救等)
   - 身份声称一致性(唯一保留的核心铁律)

[B. 你的推理框架]        ← 对所有角色通用、干净、无结论
   - 5步思考(保留,收紧)
   - 开放式盘点问题(替换掉双向先验)

[C. 你的角色]            ← 每人不同,本局内不变
   - 身份、私有信息、私有信息使用边界

[D. 当前局势(结构化)]   ← 每轮变化,放最后
   - claims表 / 投票矩阵 / 存活死亡 / 发言位次

[E. 本轮任务与输出格式]  ← 放最末(recency)
   - 发言目标、200字限制、顿号规则
```

现有 5 步思考(第 90–98 行)方向对,保留;只是挪到 B 区,并删掉与 persona 混在一起的部分。

## 四、可改进 vs 只能等模型

| 问题 | 归因 | 能否靠 prompt 解决 |
|---|---|---|
| 输出空话、两边都占一句 | 双向先验 + 强调通胀 | ✅ 完全可改(改成开放提问) |
| 发言前后矛盾 | 互斥硬规则 | ✅ 可改(消除"必须+独立"冲突) |
| 漏掉关键事实/记错谁投谁 | 散文记牌 | ✅ 可改(结构化状态) |
| 为演性格牺牲判断 | persona 混入推理 | ✅ 可改(分离) |
| 成本高 | 每轮全量重传 | ✅ 可改(缓存 + 结构化压缩) |
| 深层身份推理(多跳链) | 模型社交推理/心智理论上限 | ⚠️ prompt 能引导,天花板在模型 |
| 诈唬与反诈唬(悍跳最优、识别倒钩) | 对抗性博弈,当前模型普遍弱 | ⚠️ 主要等模型 |
| 跨轮次信念更新(无显式状态时自记忆修正) | 长上下文一致性 | ⚠️ 结构化状态可缓解,仍等模型 |

## 五、成本(prompt caching)

一次性全量塞、按轮调用,强烈建议做 **prompt caching**:
- A(规则)+ B(框架)整局甚至跨局不变,C(角色)整局不变,只有 D/E 每轮变。
- 按"稳定在前、易变在后"排序后,前缀完全可缓存,同局后续轮次输入成本大幅下降。

这比压缩历史对话省得多,且不损失信息。历史对话可更激进地换成结构化摘要(claims/投票矩阵),token 更省、模型还读得更准 —— 一举两得。
